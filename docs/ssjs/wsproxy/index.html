WSProxy Basics

Make SOAP API great again!

Simple Object Access Protocol is one of the most popular ways of for exchanging structured information. It's old. It's complex. It's hard to wrap head around quickly. Most of the modern web uses REST API (more about it in a minute). But SOAP is still alive in a lot of places. Salesforce Marketing Cloud included. Sort of.

As far as SFMC is still using SOAP API, it is in freeze mode. Not actively developed (officialy) as focus is put on extending the scope of the REST API. But right now we are in this weird transitional period, when some features of Marketing Cloud are accessible with the SOAP, some with the REST. And some with both, even when official documentation disagree (for example undocumented Automation Endpoint). When we can expect fully RESTful SFMC? Recent launch of the WSProxy should be enough for an answer. Let's dive in why.

WSProxy and SOAP#

What is WSProxy? It's a SSJS wrapper around the bad and ugly of the SOAP API. And it is absolutely wonderful. One code comparision is worth thousand words.
